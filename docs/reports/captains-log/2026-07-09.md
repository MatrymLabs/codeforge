# Captain's Log — 2026-07-09

## Summary

Built the **browser gateway** — the engine's fourth driver and the first one a
stranger can click. CodeForge is now playable in a web page over a WebSocket,
packaged for a one-click Render deploy. This closes the top hirability gap: a
live, clickable demo. Suite grew 201 → 208, all green; the production path
(real uvicorn + `websockets`) was smoke-tested end to end.

## What shipped

1. **`parts/web_gateway.py`** — a FastAPI app exposing `/` (the client),
   `/ws` (the WebSocket driver), and `/health`. It runs the same
   front-desk-then-tick ritual as the TCP gateway, but async: a single-writer
   outbound pump (`_pump`) drains a per-connection queue that doubles as the
   player's echo sink, so prompts, tick responses, and other players'
   broadcasts all leave in order. `handle_command` stays the only door.
2. **`parts/web/index.html`** — a self-contained xterm.js terminal (CDN),
   CodeForge-branded, with client-side password masking (echo `*` when the
   last line is a password prompt) and a live/disconnected status light.
3. **Demo safety** (the "demo-mode fallback" choice, mapped to a MUD): a seat
   cap and idle timeout reused from the gateway, an ephemeral DB via
   `CODEFORGE_DB`, no admin surface mounted, and `@`-verbs still rank-gated —
   a public link can't be farmed or accrete state.
4. **`codeforge web`** CLI verb (binds `$PORT`) + **`render.yaml`** blueprint
   (Docker runtime, overrides the entrypoint to `codeforge web`, ephemeral DB,
   no secrets). `websockets` added as a runtime dep; the HTML shipped as
   package-data so it survives `pip install .` in the image.
5. **Tests** (`tests/test_web_gateway.py`, over a real ASGI WebSocket): health,
   the client page, guest entry + a reachable tick, register-through-the-tick,
   graceful quit (farewell flushes before close), wrong-password reprompt, and
   the full-house turnaway.

## A bug caught by driving it, not just testing it

TestClient (ASGI) was green, but a live smoke test over real uvicorn +
`websockets` showed `quit` closing the socket **before** the farewell line
flushed — I was cancelling the outbound pump too early. Fixed teardown to drain
the queue before closing, and pinned it with
`test_quit_delivers_the_farewell_before_the_socket_closes`. Lesson reinforced:
the ASGI test harness and a real socket are not the same surface — smoke-test
the deployed path.

## Follow-up: guest access removed (login now required)

Josh caught it in live play: hitting Enter at the front desk spawned an
anonymous player. Access should require one of the two auth paths. Removed the
guest branch from **both** front desks (TCP `parts/gateway.py` and web
`parts/web_gateway.py`): an empty line now re-prompts with "Login required"
instead of seating a body, and the prompt/splash no longer advertise GUEST.
Swept the surface — splash text, the web banner, `ritual.sh`, README, and
`docs/RUNNING.md`. Tests that leaned on guest to reach the world now register an
account first (`_connect_player`); added `test_empty_enter_does_not_grant_access`
and a WS twin. Live-verified on a real socket.
