Audit → fix → feature day. Merged two long-lived auth-fix branches that had

## What shipped (in order)

1. **Audit of `main`.** Found the repo's own `CLAUDE.md` claimed "secret case
   preservation" and "telnet echo blackout" were merged, but `main` had
   neither — both lived on unmerged branches. `forge.py` still ran a blanket
   `raw.lower()` over secrets. `accounts.json` (account salt+hash) was tracked
   despite the policy to ignore it.
2. **Merged the auth fixes.** `feat/telnet-echo-mask` was a linear superset of
   `fix/secret-case-preservation` (+ the httpx2 testclient fix), so one
   `--no-ff` merge brought all three. `forge.py` now parses secrets from the
   original (un-lowercased) input; the gateway masks passwords with an
   IAC WILL/WONT ECHO blackout.
3. **Untracked `accounts.json`** (`git rm --cached` + `.gitignore`). Note: the
   old hash remains in history — rotation, not history-scrubbing, is the real
   fix if it's ever considered sensitive.
4. **Task 2 — in-game `passwd`.** New `accounts.change_password()` (verify old
   constant-time, min 4 chars, salted rehash, ORM only); a `passwd <old>
   <new> <new-again>` engine-tick branch (secrets read from original input,
   case preserved); and a gateway dialogue that drives the three prompts under
   the echo blackout. Ten new tests: success, wrong-old, mismatch, mixed-case
   roundtrip, min-length, usage, requires-account, direct-function, and two
   over-the-wire socket tests asserting the blackout bytes.
5. **Manual test + a lesson.** In-game `passwd` returned "Huh?" over the wire.
   Cause: a **ghost `spark` server (PID 5210)** from before the change was
   still bound to :4000 — a running process serves the code it imported at
   launch. Killed it, restarted from the repo root, verified the new command
   set live. Classic *deploy ≠ restart*.
6. **Housekeeping.** Pruned two throwaway test accounts from the local DB
   (only `matrym`/`matlabs` remain); deleted four merged branches.
7. **DB path hardening.** `parts/db.py` `DB_PATH` was cwd-relative — launching
   from the wrong directory silently created a second, empty `codeforge.db`.
   Now an absolute default anchored to the repo root via `_default_db_path()`,
   overridable with `CODEFORGE_DB`. Closed the backlog item; pinned with tests.
8. **Security review + hardening pass (this branch).** Reviewed the engine
   against MUD-server security best practices (see
   [`../security/security-roadmap.md`](../security/security-roadmap.md)) and
   began implementing the code-able items.

## Lessons banked

- **Deploy ≠ restart.** A live server is a snapshot of code at launch. When a
  fix "doesn't take," suspect a ghost process before suspecting the code:
  `ss -ltnp | grep :4000` / `lsof -i :4000`, kill, restart from the repo root.
- **A repo that breaks its own stated rules reads worse than one with none.**
  `CLAUDE.md` described features that weren't on `main`. Docs and `main` must
  agree; "merged" must mean merged.
- **`.gitignore` does not untrack.** A file already committed keeps being
  tracked until explicitly removed from the index.
