# Reports

Working records for CodeForge - the story behind the commits.

This directory holds the human-readable narrative that a git history alone
doesn't tell: what was decided and why, what was learned, and where the
project is headed.

## Layout

| Path | What lives here |
|------|-----------------|
| [`captains-log/`](captains-log/) | Dated daily logs - one file per working day (`YYYY-MM-DD.md`). What happened, what broke, what shipped. |
| [`professional-development/`](professional-development/) | Skills demonstrated, lessons banked, and growth goals. The "what am I getting better at" ledger. |
| [`security/`](security/) | Security reference material and the hardening roadmap for the engine. |

## Conventions

- **One captain's log per day.** Name it `captains-log/YYYY-MM-DD.md`. Keep it
  terse and factual: a stranger should be able to reconstruct the day.
- **Dates are absolute.** No "yesterday" / "last week" - write the date.
- **Link to commits and files** (`file.py:line`, short SHAs) so the log stays
  anchored to the code it describes.
- These are portfolio artifacts. Assume an interviewer will read them.
